# Claude Code 核心流程

## 核心流程概述

Claude Code 的核心流程从命令行入口开始，经过多个阶段最终完成与大模型的交互，主要包括以下几个关键步骤：

## 1. 入口与初始化（src/entrypoints/cli.tsx）

**主要职责：**
- 处理命令行参数和快速路径
- 支持多种运行模式：--version、--daemon、--bridge、--remote-control 等
- 初始化配置和状态

**关键流程：**
```typescript
// 简化版流程
function main() {
  // 1. 解析命令行参数
  // 2. 处理快速路径（如 --version、--daemon 等）
  // 3. 加载主程序
  import('./main.tsx').then(mainModule => {
    mainModule.default()
  })
}
```

## 2. CLI 定义与主循环（src/main.tsx）

**主要职责：**
- 使用 Commander.js 定义 CLI 命令
- 管理会话状态和权限
- 分发到 REPL 或 Headless 模式

**关键流程：**
```typescript
// 简化版流程
program
  .command('chat')
  .action(async () => {
    // 1. 初始化应用状态
    const appState = createAppState()
    // 2. 创建查询引擎
    const queryEngine = new QueryEngine(appState)
    // 3. 启动 REPL 或 Headless 模式
    if (isInteractive) {
      startREPL(queryEngine)
    } else {
      await runHeadless(queryEngine)
    }
  })
```

## 3. 核心查询引擎（src/query.ts）

**主要职责：**
- 管理对话轮次循环
- 处理流式响应
- 执行工具调用
- 管理上下文和状态

**关键流程：**
```typescript
export async function* query(params: QueryParams) {
  // 1. 初始化状态
  let state = {
    messages: params.messages,
    toolUseContext: params.toolUseContext,
    // ...其他状态
  }
  
  // 2. 主循环
  while (true) {
    // 3. 准备请求（系统提示、上下文、工具等）
    const request = prepareRequest(state)
    
    // 4. 调用 API
    const response = await callAPI(request)
    
    // 5. 处理响应
    yield processResponse(response)
    
    // 6. 检查是否需要继续（工具调用、多轮对话等）
    if (needsContinuation(response)) {
      state = updateState(state, response)
      continue
    }
    break
  }
}
```

## 4. API 调用层（src/services/api/claude.ts）

**主要职责：**
- 构建 API 请求参数
- 处理流式响应
- 支持多提供商（Anthropic、Bedrock、Vertex、Azure）

**关键流程：**
```typescript
export async function* callModel(options) {
  // 1. 构建请求参数
  const params = buildRequestParams(options)
  
  // 2. 调用 Anthropic SDK 流式端点
  const stream = await anthropic.messages.stream(params)
  
  // 3. 处理流式事件
  for await (const event of stream) {
    yield processStreamEvent(event)
  }
}
```

## 5. 交互式界面（src/screens/REPL.tsx）

- 处理用户输入
- 显示消息和工具输出
- 管理权限提示

## 完整交互流程

1. **启动**：用户运行 `claude` 命令
2. **初始化**：CLI 解析参数，加载配置
3. **会话创建**：创建应用状态和查询引擎
4. **用户输入**：REPL 界面接收用户消息
5. **请求准备**：构建 API 请求（消息、系统提示、工具等）
6. **API 调用**：发送请求到 Anthropic API
7. **流式响应**：逐块接收和处理模型响应
8. **工具执行**：如果模型调用工具，执行相应操作
9. **结果展示**：在终端显示响应和工具输出
10. **循环**：等待下一个用户输入

## 关键特性

- **流式处理**：实时显示模型响应
- **工具集成**：支持 61+ 种工具调用
- **上下文管理**：自动管理对话历史和上下文
- **多轮对话**：支持连续交互和工具调用
- **状态持久化**：会话状态和工具权限管理

这个架构设计确保了高效、交互式的 CLI 体验，同时保持了模块化和可扩展性。
